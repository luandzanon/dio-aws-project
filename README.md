# Desafio de Projeto: Otimizando Custos na AWS

## Contexto
Este projeto faz parte do **Bootcamp Santander**, focando na prática de arquitetura em nuvem e gestão de custos (FinOps) na AWS.

## O Cenário
Uma grande empresa farmacêutica que atua no modelo B2B (distribuição para farmácias e hospitais) está enfrentando custos elevados em sua infraestrutura de TI on-premise e busca migrar parte de sua operação para a nuvem AWS para otimizar despesas.

### Os Desafios da Empresa:
1.  **Armazenamento Crescente:** A empresa precisa guardar históricos de notas fiscais e registros regulatórios por 5 anos, gerando um custo alto de armazenamento em discos físicos que raramente são acessados.
2.  **Ociosidade de Servidores:** O sistema de pedidos das farmácias tem picos de acesso apenas em horários comerciais, mas os servidores atuais ficam ligados 24/7, gastando energia e manutenção desnecessariamente.
3.  **Falta de Visibilidade:** A diretoria não consegue prever os gastos mensais de TI e frequentemente é surpreendida com despesas de manutenção de hardware.

## O Objetivo
O objetivo deste projeto é criar uma proposta técnica descrevendo **3 serviços da AWS** que resolvam diretamente os problemas de custo citados acima.

Para cada serviço escolhido, o projeto deve entregar:
1.  **Nome da ferramenta.**
2.  **Foco da ferramenta.**
3.  **Descrição de caso de uso.**

## Entregáveis
O repositório deve conter arquivos `.md` detalhando as soluções arquitetadas para este case.
